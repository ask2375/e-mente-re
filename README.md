# 設定

## １．『Live Sass Compiler （v6.1.2）』をインストール

※『Live Sass Compiler』は 2 種類あります。
　必ず Glenn Marks 氏の『Live Sass Compiler』をご使用ください。最新版へアップロードしてください。
　 Ritwick Dey 氏の『Live Sass Compiler』は Dart Sass 非対応のためコンパイルできません。

## ２．『Live Sass Compiler』の設定をする

### ※ワークスペースで既に設定しているため、基本的には 2 と 3 の手順は不要です。

### 『.vscode』フォルダの中の setting.json に設定を保存しています。

（1）プラグイン名の右下にある『歯車のボタン』をクリックして『拡張機能の設定』をクリック
（2）設定を 3 箇所行う
①「Disable Source Map」にチェックを入れる
└ ソースマップファイルの生成を無効にする場合はチェック。
納品時には不要なファイルですが、無いとデベロッパーツールなどでコードを探す時に不便なのであった方が良い。

②「Disable Auto Prefixer」のチェックが入っていれば外す
└ 自動プレフィックスコンパイルを有効にしたいので『チェックなし』で使用する。

③「Output Format」のコンパイラの出力形式を『cssonly』にする
└ コンパイラの出力形式を選択します。
デフォルトの『both』では『CSS ファイル(.css)』と『CSS の圧縮ファイル(.min.css)』が両方出力されますが、
CSS ファイルのみの『cssonly』で良い。

（3）「Target Directory」に『assets/css/』と記述する
└ assets フォルダの中に images や js フォルダを格納する場合。
└ images フォルダを assets フォルダ内に格納しない場合は『css/』で良い。

## 3．「cmd + shift + p」で、「Reload Window」を入力し、VSCode をリロードさせる。

└ リロードしないと設定が反映されない。

## パーシャルファイル(\_(アンダーバー)●●.scss)の作成について

①『module』や『page』フォルダにパーシャルファイルを作成していきますが、
\_index.scss に作成したファイルの名前を必ず追加する。
\_about.scss を作成したのであれば、「@forward 'about';」と記載する。

② パーシャルファイルを作成したら、1 行目に『@use '../global' as \*;』 ← 「\」は不要(readme だと追加されてしまいます。)

## 仕様

- PC ファーストが前提の仕様です。
- rem 記述を前提としています。
  例えば、デザインカンプで 16px と指定があったら、rem(16)と記載すると 16px でコンパイルされる。
- ルートフォントを vw で設定していることから、
  PC サイズのレイアウトをタブレットで表示させることが出来ます（rem で書いた場合のみ）。
